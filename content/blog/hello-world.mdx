The first post on any blog is a ritual. You feel the weight of the blank page, the cursor blinking at you like it's asking, "So, what do you actually have to say?"

I've been writing code for over a decade, but writing _about_ code is a different muscle. It requires you to slow down enough to observe your own thinking—which is harder than it sounds when you're used to moving fast and breaking things.

## Why now?

I spent years consuming other people's writing without contributing back. Technical blogs, conference talks, open-source documentation—they shaped how I think about software. It's time to close that loop.

This space is for:

- **Deep dives** into systems I've built (and the ones I've broken)
- **Opinions** on tooling, architecture, and the craft of engineering
- **Honest retrospectives** on decisions that seemed right at the time
- **Explorations** of technologies I'm learning, documented in real-time

## What you won't find here

Hot takes for engagement. Listicles disguised as insight. Advice from someone pretending to have it all figured out.

I'm not interested in being a thought leader. I'm interested in being a thoughtful practitioner who shares their work.

## The technical stack

This site is built with Next.js 16, TypeScript, Tailwind CSS, and MDX. It's statically exported and deployed as a collection of HTML files—no server, no database, no complexity beyond what's necessary.

The code is open source. If you're curious how something works, [check the repo](https://github.com).

Let's see where this goes.
